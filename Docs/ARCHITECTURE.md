# Architecture

## 1. Обзор

Проект состоит из:

- Telegram API слой
- Application Logic слой
- Infrastructure слой (хранилище, логирование, конфиг)

## 2. Компоненты

### 2.1 Bot Handlers

- Принимают апдейты из Telegram
- Валидируют входные данные
- Делегируют в сервисный слой

### 2.2 Services

- Бизнес-логика сценариев
- Работа с состоянием пользователя

### 2.3 Repositories (опционально)

- Абстракция для БД/файлового хранилища

## 3. Поток запроса

1. Update из Telegram
2. Router определяет handler
3. Handler вызывает service
4. Service формирует ответ
5. Ответ отправляется пользователю

## 4. Ошибки и логирование

- Глобальный обработчик исключений
- Структурированные логи
- Корреляция логов по `user_id`
