# Functional and Non-Functional Requirements

## 1. Функциональные требования

### 1.1 Базовые

- Бот должен отвечать на команду `/start`.
- Бот должен отвечать на команду `/help`.
- Бот должен обрабатывать неизвестные команды.

### 1.2 Расширенные

- Бот должен поддерживать хранение состояния пользователя.
- Бот должен иметь админ-команды (опционально).

## 2. Нефункциональные требования

- **Производительность:** среднее время ответа до 1 секунды.
- **Надежность:** корректная обработка ошибок API Telegram.
- **Безопасность:** токены не хранятся в репозитории.
- **Наблюдаемость:** логирование входящих апдейтов и ошибок.

## 3. Ограничения

- Работа только в Telegram.
- Язык интерфейса: уточнить.

## 4. Критерии приемки

- [ ] `/start` и `/help` работают стабильно
- [ ] Ошибки не падают в необработанное исключение
- [ ] Есть базовое покрытие тестами
